## ✨ 기능 특징

### 🔌 다중 프록시 프로토콜 지원
- **HTTP** - 전통적인 HTTP 프록시
- **HTTPS** - 보안 HTTPS 프록시
- **SOCKS5** - TCP/UDP를 지원하는 SOCKS5 프록시
- **SOCKS4** - 레거시 SOCKS4 프록시 호환성

### 🔄 세 가지 프록시 모드

| 모드 | 설명 |
|------|------|
| **비활성화** | 프록시 비활성화, 시스템 기본 네트워크 연결 사용 |
| **수동** | 목록에서 프록시를 수동으로 선택 |
| **자동** | URL 규칙에 따라 자동으로 일치하는 프록시 선택 (PAC 모드) |

| ![](../../public/img/demo-popup-01.png) | ![](../../public/img/demo-popup-02.png) | ![](../../public/img/demo-popup-03.png) |
|:---:|:---:|:---:|
| 비활성화 모드 | 수동 모드 | 자동 모드 |

### 📋 유연한 URL 규칙 구성

- **프록시를 사용하지 않는 주소** (`bypass_urls`): 직접 연결할 도메인/IP
- **프록시를 사용하는 주소** (`include_urls`): 프록시 접근이 필요한 도메인
- 와일드카드 `*` 및 도메인 매칭 지원
- 서로 다른 웹사이트에 서로 다른 프록시를 사용하는 시나리오에 적합

### 🔐 프록시 인증 지원

- 사용자명/비밀번호 인증 지원
- 프록시 서버의 인증 요청 자동 처리
- 자격 증명 정보 안전하게 저장

### 🧪 프록시 테스트 기능

- **연결 테스트**: 프록시 사용 가능 여부 확인
- **지연 시간 측정**: 프록시 응답 시간 테스트
- **일괄 테스트**: 모든 프록시를 한 번에 테스트
- **색상 표시**: 초록색(<500ms) / 주황색(≥500ms) / 빨간색(실패)

### 🌙 테마 모드

- **라이트 모드**: 주간 사용
- **다크 모드**: 야간 사용
- **자동 전환**: 시간에 따라 테마 자동 전환

## 🚀 빠른 시작

### 확장 프로그램 설치

1. Chrome을 열고 `chrome://extensions/`로 이동합니다.
2. 오른쪽 상단의 **"개발자 모드"**를 활성화합니다.
3. **"압축 해제된 확장 프로그램 로드"**를 클릭합니다.
4. `ProxyAssistant/src` 폴더를 선택합니다.

### 프록시 추가

1. 확장 프로그램 아이콘을 클릭하여 팝업을 엽니다.
2. **"설정"** 버튼을 클릭하여 설정 페이지를 엽니다.
3. **"새로 추가"** 버튼을 클릭하여 새 프록시를 추가합니다.
4. 프록시 정보를 입력합니다:
   - 프록시 이름
   - 프로토콜 유형 (HTTP/HTTPS/SOCKS5)
   - 프록시 주소 (IP 또는 도메인)
   - 포트 번호
   - (선택 사항) 사용자名과 비밀번호
5. **"저장"** 버튼을 클릭합니다.

### 프록시 사용

**수동 모드**:
1. 팝업에서 **"수동"** 모드를 선택합니다.
2. 목록에서 프록시를 선택합니다.
3. "연결됨" 상태가 표시되면 활성화된 것입니다.

**자동 모드**:
1. 팝업에서 **"자동"** 모드를 선택합니다.
2. 설정 페이지에서 각 프록시에 대한 URL 규칙을 구성합니다.
3. 방문하는 웹사이트에 따라 프록시가 자동으로 선택됩니다.

## 📖 상세 설명

### URL 규칙 구문

다음과 같은 매칭 규칙을 지원합니다:

```
# 정확한 일치
google.com

# 하위 도메인 일치
.google.com
www.google.com

# 와일드카드 일치
*.google.com
*.twitter.com

# IP 주소
192.168.1.1
10.0.0.0/8
```

### PAC 스크립트 자동 모드

자동 모드는 PAC (Proxy Auto-Config) 스크립트를 사용합니다:
- 현재 URL에 따라 프록시를 자동으로 선택합니다.
- 폴백 정책 지원 (직접 연결 또는 거부)
- 브라우저 시작 시 마지막 구성 자동 복원

### 바로 가기 키

| 작업 | 방법 |
|------|------|
| 프록시 카드 펼치기/접기 | 카드 헤더 클릭 |
| 모든 카드 펼치기/접기 | "모두 펼치기" 버튼 클릭 |
| 프록시 드래그하여 정렬 | 카드 헤더의 드래그 핸들 드래그 |
| 비밀번호 표시/숨기기 | 비밀번호 필드의 눈 아이콘 클릭 |
| 단일 프록시 테스트 | "테스트" 버튼 클릭 |
| 모든 프록시 테스트 | "모두 테스트" 버튼 클릭 |

### 구성 가져오기/내보내기

1. **구성 내보내기**: "구성 내보내기"를 클릭하여 JSON 파일을 다운로드합니다.
2. **구성 가져오기**: "구성 가져오기"를 클릭하고 복원할 JSON 파일을 선택합니다.

구성에 포함된 내용:
- 모든 프록시 정보
- 테마 설정
- 동기화 설정

## 🔧 기술 아키텍처

### Manifest V3

- Chrome 확장 프로그램 Manifest V3 사양 사용
- Service Worker가 백그라운드 페이지 대체
- 더 안전하고 효율적인 아키텍처

### 핵심 모듈

1. **service-worker.js**:
   - 프록시 구성 관리
   - PAC 스크립트 생성
   - 인증 처리
   - 프록시 테스트 로직

2. **popup.js**:
   - 팝업 인터페이스 상호작용
   - 프록시 상태 표시
   - 빠른 프록시 전환

3. **main.js**:
   - 설정 페이지 로직
   - 프록시 관리 (CRUD)
   - 드래그 앤 드롭 정렬
   - 가져오기/내보내기

4. **i18n.js**:
   - 다국어 지원
   - 실시간 언어 전환

### 데이터 저장

- `chrome.storage.local`: 로컬 저장
- `chrome.storage.sync`: 클라우드 동기화 저장
- 저장소 할당량 자동 처리

## 📝 사용 시나리오

### 시나리오 1: 다중 프록시 전환

- 서로 다른 네트워크 환경에 대해 서로 다른 프록시 구성
- 사무실 네트워크에는 회사 프록시 사용
- 가정 네트워크에는 VPN 프록시 사용
- 빠른 원클릭 전환

### 시나리오 2: 스마트 라우팅

- 국내 웹사이트는 직접 연결
- 특정 웹사이트는 프록시 통해
- 도메인에 따른 자동 선택

### 시나리오 3: 프록시 풀 테스트

- 여러 프록시 가져오기
- 일괄 지연 시간 테스트
- 최적의 프록시 선택

### 시나리오 4: 팀 공유

- 구성 파일 내보내기
- 팀원과 공유
- 통합 프록시 구성

## ⚠️ 주요 사항

1. **권한 설명**: 확장에 다음 권한이 필요합니다:
   - `proxy`: 프록시 설정 관리
   - `storage`: 구성 저장
   - `webRequest`: 인증 요청 처리
   - `<all_urls>`: 모든 웹사이트 URL 접근

2. **다른 확장 프로그램과의 충돌**: 프록시 충돌이 발생하면 다른 프록시 확장 프로그램을 비활성화하세요.

3. **보안**: 자격 증명은 브라우저에 로컬로 저장됩니다. 기기의 보안을 확인하세요.

4. **네트워크 요구 사항**: 프록시 서버에 접근할 수 있는지 확인하세요.
