# GitHub CI 实施方案

> **供代理执行参考：** 实施本方案时，必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 逐任务执行，步骤使用复选框 `- [ ]` 追踪状态。

**目标：** 为仓库增加 GitHub Actions CI，在 `main` 分支 `push` 和所有 `pull_request` 上自动执行，并拆分为单元测试、集成测试、e2e 测试和构建四个独立 job。

**方案：** 在 `.github/workflows/ci.yml` 中新增一个工作流，统一运行在 `ubuntu-latest`。单元测试始终执行；集成测试和 e2e 测试先检测仓库中是否存在对应测试文件，不存在时显式跳过，避免当前测试结构导致 CI 误报失败；构建 job 安装 `web-ext`，通过现有 `make build` 和 `script/build.sh` 产出扩展包，并上传构建产物供审查下载。

**技术栈：** GitHub Actions、Node.js 20、npm、Make、Jest、现有 `script/build.sh`

### 任务 1：新增 GitHub CI 工作流

**涉及文件：**
- 新增：`.github/workflows/ci.yml`

- [ ] **步骤 1：定义触发条件、权限和并发控制**

```yaml
name: CI

on:
  push:
    branches:
      - main
  pull_request:

permissions:
  contents: read

concurrency:
  group: ci-${{ github.workflow }}-${{ github.event.pull_request.number || github.ref }}
  cancel-in-progress: true
```

- [ ] **步骤 2：添加单元测试 job**

```yaml
jobs:
  unit:
    name: Unit Tests
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Run unit tests
        run: npm run test:unit -- --no-cache
```

- [ ] **步骤 3：添加集成测试和 e2e 测试 job，并显式处理空测试目录**

```yaml
  integration:
    name: Integration Tests
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Detect integration tests
        id: detect
        run: |
          if find tests/integration -name '*.test.js' -print -quit 2>/dev/null | grep -q .; then
            echo 'has_tests=true' >> "$GITHUB_OUTPUT"
          else
            echo 'has_tests=false' >> "$GITHUB_OUTPUT"
          fi

      - name: Run integration tests
        if: steps.detect.outputs.has_tests == 'true'
        run: npm run test:integration -- --no-cache

      - name: Skip integration tests
        if: steps.detect.outputs.has_tests != 'true'
        run: echo 'No integration tests found. Skipping.'

  e2e:
    name: E2E Tests
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Detect e2e tests
        id: detect
        run: |
          if find tests/e2e -name '*.test.js' -print -quit 2>/dev/null | grep -q .; then
            echo 'has_tests=true' >> "$GITHUB_OUTPUT"
          else
            echo 'has_tests=false' >> "$GITHUB_OUTPUT"
          fi

      - name: Run e2e tests
        if: steps.detect.outputs.has_tests == 'true'
        run: npm run test:e2e -- --no-cache

      - name: Skip e2e tests
        if: steps.detect.outputs.has_tests != 'true'
        run: echo 'No e2e tests found. Skipping.'
```

- [ ] **步骤 4：添加构建 job 和产物上传**

```yaml
  build:
    name: Build Extension
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Install web-ext
        run: npm install -g web-ext

      - name: Build extension packages
        run: make build VERSION=ci-${{ github.run_number }}

      - name: Upload build artifacts
        uses: actions/upload-artifact@v4
        with:
          name: proxy-assistant-build-${{ github.run_number }}
          path: build/ProxyAssistant_*
          if-no-files-found: error
```
